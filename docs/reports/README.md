# Verification Reports

This folder stores outputs generated by `verify-gemschi`.

## Naming

- Primary format: `verification-YYYY-MM-DD.md`
- Same-day rerun: `verification-YYYY-MM-DD-v2.md`, `-v3.md`, ...

## Required Sections

Each report must include:

1. `# Verification Report - YYYY-MM-DD`
2. `## Scope`
3. `## Summary`
4. `## Findings` (all checklist items must have one status)
5. `## Main Agent Worklist`
6. `## Open Verification Gaps`

## Status Values

- `Verified`
- `PartiallyVerified`
- `NotVerified`
- `Unknown`

## Evidence Rule

Do not mark an item as `Verified` without direct evidence from code or tests.
